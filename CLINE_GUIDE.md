# 🤖 Руководство по работе с Cline

## Что такое Cline?

**Cline** — это AI-ассистент для разработки, интегрированный в Visual Studio Code. Я (Cline) помогаю вам:
- Писать и редактировать код
- Создавать файлы и структуры проектов
- Выполнять команды в терминале
- Искать информацию в интернете
- Анализировать и рефакторить код
- Отвечать на вопросы по программированию

## 🎯 Основные возможности

### 1. **Работа с файлами**
- ✅ Создание новых файлов
- ✅ Чтение существующих файлов
- ✅ Редактирование кода (точечные изменения)
- ✅ Поиск по файлам (regex)
- ✅ Просмотр структуры проекта

### 2. **Выполнение команд**
- ✅ Запуск CLI команд (git, npm, pip, docker и т.д.)
- ✅ Установка пакетов
- ✅ Запуск серверов разработки
- ✅ Выполнение тестов

### 3. **Веб-поиск**
- ✅ Поиск актуальной информации
- ✅ Чтение документации
- ✅ Поиск решений проблем

### 4. **Анализ кода**
- ✅ Просмотр определений функций/классов
- ✅ Анализ структуры проекта
- ✅ Рефакторинг кода

## 📝 Как работать с Cline

### Режимы работы

#### **ACT MODE** (Режим действия)
В этом режиме я выполняю задачи напрямую:
```
Вы: "Создай FastAPI приложение с подключением к PostgreSQL"
Я: [создаю файлы, пишу код, настраиваю проект]
```

#### **PLAN MODE** (Режим планирования)
В этом режиме мы сначала обсуждаем план:
```
Вы: "Мне нужно создать сложную систему..."
Я: "Давайте спланируем архитектуру..."
[обсуждение]
Вы: "Переключись в ACT MODE"
Я: [начинаю реализацию]
```

### Примеры запросов

#### ✅ Хорошие запросы:
```
"Создай Django проект с PostgreSQL и Docker"
"Добавь эндпоинт для регистрации пользователя"
"Оптимизируй этот SQL запрос"
"Напиши тесты для функции calculate_total"
"Исправь ошибку в файле main.py"
```

#### ❌ Неясные запросы:
```
"Сделай что-нибудь"
"Исправь"
"api key или fontier" (непонятно что нужно)
```

### Как правильно формулировать задачи

1. **Будьте конкретны:**
   ```
   ❌ "Создай API"
   ✅ "Создай FastAPI приложение с эндпоинтами для CRUD операций с пользователями"
   ```

2. **Указывайте технологии:**
   ```
   ❌ "Подключи базу данных"
   ✅ "Подключи PostgreSQL через SQLAlchemy в Django проекте"
   ```

3. **Уточняйте детали:**
   ```
   ❌ "Добавь авторизацию"
   ✅ "Добавь JWT авторизацию с использованием библиотеки PyJWT"
   ```

## 🔧 Практические примеры

### Пример 1: Создание Django проекта
```
Вы: "Создай Django проект с PostgreSQL, Docker и REST API"

Я создам:
- requirements.txt с зависимостями
- settings.py с настройками БД
- docker-compose.yml
- Dockerfile
- .env.example для конфигурации
- README.md с инструкциями
```

### Пример 2: Добавление функционала
```
Вы: "Добавь модель User с полями email, username, created_at"

Я:
1. Создам модель в models.py
2. Создам миграции
3. Добавлю сериализаторы (если REST API)
4. Создам views и urls
```

### Пример 3: Отладка
```
Вы: "В файле main.py ошибка на строке 45, исправь"

Я:
1. Прочитаю файл
2. Найду ошибку
3. Предложу исправление
4. Внесу изменения
```

## 💡 Полезные советы

### 1. **Используйте task_progress**
Я автоматически отслеживаю прогресс выполнения задачи:
```
- [x] Создан Django проект
- [x] Настроен PostgreSQL
- [ ] Добавлены модели
- [ ] Созданы API эндпоинты
- [ ] Написаны тесты
```

### 2. **Задавайте уточняющие вопросы**
Если что-то непонятно, я спрошу:
```
Я: "Какую версию Django использовать: 4.2 или 5.0?"
Вы: "Django 5.0"
```

### 3. **Проверяйте результаты**
После выполнения команд я покажу результат:
```
Я: "Выполнил: pip install django"
[показываю вывод команды]
```

### 4. **Используйте контекст**
Я вижу структуру вашего проекта и могу работать с существующими файлами:
```
Вы: "Добавь новый эндпоинт в существующий FastAPI проект"
Я: [анализирую структуру, нахожу main.py, добавляю эндпоинт]
```
