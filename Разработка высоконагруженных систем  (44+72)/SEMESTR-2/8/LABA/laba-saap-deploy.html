🚂 Деплой на Railway

Railway — платформа для деплоя приложений прямо с GitHub.

Шаг 0: Как Railway узнаёт, как запускать приложение

В проекте используется railway.json (а не Procfile). В корне файл с полем deploy.startCommand:

"startCommand": "uvicorn main:app --host 0.0.0.0 --port $PORT"

$PORT — переменная окружения, которую Railway выделяет приложению.

Шаг 1: Создать аккаунт на Railway

  1. Перейдите на railway.app
  2. Нажмите Start New Project
  3. Выберите Deploy from GitHub Repo
  4. Авторизуйте GitHub (дайте Railway доступ к репозиториям)

Шаг 2: Выбрать репозиторий и ветку

  1. Выберите свой репозиторий
  2. Выберите ветку main
  3. Railway автоматически обнаружит, что это Python-проект

Шаг 3: Добавить requirements.txt (важно!)

Railway нужно знать, какие зависимости установить. В корне должен быть requirements.txt:

fastapi==0.110.0 uvicorn==0.28.0 pydantic==2.8.0 pytest==7.4.4 httpx==0.25.2 black==24.10.0

Шаг 4: Добавить переменные окружения (опционально)

На странице сервиса приложения в Railway: Variables → добавьте переменные при необходимости.

Шаг 5: Дождитесь деплоя

  1. Railway автоматически запустит деплой
  2. Включит приложение на публичный URL
  3. Вы увидите что-то вроде: https://your-app-xxxx.railway.app

Проверка

После деплоя:

  • Приложение: https://your-app-xxxx.railway.app/health
  • Swagger UI: https://your-app-xxxx.railway.app/docs

Автоматический деплой

После первого деплоя: каждый push в main автоматически обновит приложение на Railway.

🔄 Деплой через GitHub Actions (deploy.yml)

Используется единый .github/workflows/deploy.yml:

  • Job test: checkout → Python 3.13 → pip install → Init PostgreSQL → pytest
  • Job deploy: зависит от test → Node.js → Railway CLI → railway up --service web
- name: Deploy to Railway run: | railway up --service ${{ secrets.RAILWAY_SERVICE || 'web' }} env: RAILWAY_TOKEN: ${{ secrets.RAILWAY_TOKEN }}

Секреты в GitHub (Settings → Secrets and variables → Actions)

  • RAILWAY_TOKEN (обязательно) — токен с Railway: Settings → Tokens → Create Token
  • RAILWAY_SERVICE (необязательно) — имя сервиса. По умолчанию web

Важно: аргумент --service обязателен. Проект и сервис в Railway — разные сущности.

3.3 Изменения в main.py

import psycopg from psycopg.rows import dict_row DB_CONFIG = { "host": os.getenv("DB_HOST", "localhost"), "port": int(os.getenv("DB_PORT", "5432")), "user": os.getenv("DB_USER", "app"), "password": os.getenv("DB_PASSWORD", "app"), "dbname": os.getenv("DB_NAME", "eshop"), } def get_db_connection(): return psycopg.connect(**DB_CONFIG, row_factory=dict_row) @app.get("/products") async def get_products(): conn = get_db_connection() try: with conn.cursor() as cur: cur.execute("SELECT id, name, price, description, created_at FROM video_cards ORDER BY id") return [{**dict(r), "price": float(r["price"])} for r in cur.fetchall()] finally: conn.close()

3.4 Обновление workflow (.github/workflows/deploy.yml)

При переходе на PostgreSQL в CI нужно:

  1. Сервис Postgres — job поднимает контейнер с БД
  2. Переменные окружения — DB_HOST, DB_PORT, DB_USER, DB_PASSWORD, DB_NAME
  3. Init PostgreSQL — выполнить scripts/init_db.sql до тестов
services: postgres: image: postgres:17 env: POSTGRES_USER: app POSTGRES_PASSWORD: app POSTGRES_DB: eshop ports: - 5432:5432 options: >- --health-cmd "pg_isready -U app -d eshop" --health-interval 5s --health-timeout 5s --health-retries 5 env: DB_HOST: 127.0.0.1 DB_PORT: 5432 DB_USER: app DB_PASSWORD: app DB_NAME: eshop

Шаг Init PostgreSQL

Упрощённый вариант (через psql):

- name: Init PostgreSQL run: | docker run --rm --network host -e PGPASSWORD=app -v $PWD/scripts:/scripts postgres:17 \ psql -h 127.0.0.1 -U app -d eshop -f /scripts/init_db.sql

Порядок шагов: Checkout → Setup Python → Install dependencies → Init PostgreSQL → Black → Syntax check → Import app → Run tests.

✏️ 3.5 Самостоятельная часть

Замените все эндпоинты в main.py на вызов хранимых функций PostgreSQL (SQL или PL/pgSQL).

🚀 3.6 Job deploy — деплой на Railway

После успешного job test запускается job deploy. БД на Railway настраивается отдельно.

deploy: name: Deploy to Railway needs: test runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - name: Setup Node.js uses: actions/setup-node@v4 with: node-version: '18' - name: Install Railway CLI run: npm install -g @railway/cli - name: Deploy to Railway run: | railway up --service ${{ secrets.RAILWAY_SERVICE || 'web' }} env: RAILWAY_TOKEN: ${{ secrets.RAILWAY_TOKEN }}

Шаги: Checkout → Setup Node.js (Railway CLI через npm) → Install Railway CLI → Deploy.

🔧 3.7 Настройка БД на Railway

Шаг 1: Добавить PostgreSQL в проект

  1. Откройте проект в railway.app
  2. New → Database → PostgreSQL
  3. Дождитесь статуса готовности

Шаг 2: Связать БД с сервисом приложения

В сервисе приложения (web) добавьте переменные:

ПеременнаяЗначение
DB_HOSTхост из Postgres (например monolith.proxy.rlwy.net)
DB_PORTпорт из Variables Postgres
DB_USERпользователь
DB_PASSWORDпароль из Variables Postgres
DB_NAMEимя БД (например railway)

Шаг 3: Создать таблицу и данные

Выполните scripts/init_db.sql на прод-БД:

  • Вариант A: Postgres → Data/Query → вставьте содержимое init_db.sql
  • Вариант B: psql -h <DB_HOST> -p <DB_PORT> -U <DB_USER> -d <DB_NAME> -f scripts/init_db.sql

Проверка

https://your-app.railway.app/products — должен вернуть JSON со списком видеокарт. 500/503 — проверьте переменные и таблицу video_cards.
