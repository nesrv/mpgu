# Слайд 1: Что такое деплой?

**Деплой (Deploy)** — размещение приложения на сервере, чтобы оно стало доступно пользователям через интернет.

```
Локальный компьютер          Сервер (VPS)           Пользователи
┌──────────────┐            ┌──────────────┐        ┌──────────┐
│   Ваш код    │  ──────▶  │  Приложение  │  ◀──── │ Браузеры │
│   main.py    │   SSH      │   работает   │  HTTP  │          │
└──────────────┘            └──────────────┘        └──────────┘
```

**VPS (Virtual Private Server)** — виртуальный сервер в дата-центре с публичным IP-адресом.

# Слайд 8: Передача файлов (SFTP)

**SFTP** — безопасная передача файлов через SSH.

**В MobaXterm:**

- Левая панель автоматически показывает файлы сервера
- Drag & Drop для загрузки файлов

**В командной строке:**

```bash
# Копирование файла на сервер
scp main.py root@81.90.182.174:/opt/myapi/

# Копирование папки на сервер
scp -r myproject/ root@81.90.182.174:/opt/

# Копирование файла с сервера
scp root@81.90.182.174:/opt/myapi/main.py ./
```

**Альтернативы:**

- WinSCP (Windows)
- FileZilla (кроссплатформенный)
- Встроенный SFTP в MobaXterm

# Слайд 10: Права доступа (chmod)

В Linux каждый файл имеет права доступа для трёх категорий:


| Категория          | Обозначение |
| ------------------ | ----------- |
| Владелец (user)    | `u`         |
| Группа (group)     | `g`         |
| Остальные (others) | `o`         |


**Три типа прав:**


| Право                | Буква | Число |
| -------------------- | ----- | ----- |
| Чтение (read)        | `r`   | 4     |
| Запись (write)       | `w`   | 2     |
| Выполнение (execute) | `x`   | 1     |


---

# Слайд 11: chmod — примеры

```bash
# Сделать скрипт исполняемым
chmod +x deploy.sh

# Числовой формат: rwx = 4+2+1 = 7
chmod 755 deploy.sh    # rwxr-xr-x (владелец всё, остальные читают/выполняют)
chmod 644 main.py      # rw-r--r-- (владелец пишет, остальные только читают)
chmod 600 secrets.env  # rw------- (только владелец читает/пишет)
```

**Расшифровка 755:**

- `7` (владелец) = 4+2+1 = rwx
- `5` (группа) = 4+0+1 = r-x
- `5` (остальные) = 4+0+1 = r-x

# Слайд 13: Systemd — автозапуск сервисов

**Systemd** — система управления сервисами в Linux.

**Проблема:** Если запустить `uvicorn` в терминале и закрыть SSH — приложение остановится.

**Решение:** Создать systemd-сервис.

```
┌─────────────────┐     управляет      ┌─────────────────┐
│    systemd      │ ──────────────────▶│  uvicorn        │
│  (init система) │                    │  (приложение)   │
└─────────────────┘                    └─────────────────┘
        │
        ├── Автозапуск при загрузке ОС
        ├── Перезапуск при падении
        └── Логирование (journalctl)
```

---

# Слайд 14: Создание systemd-сервиса

```bash
# /etc/systemd/system/myapi.service

[Unit]
Description=My FastAPI App
After=network.target

[Service]
User=root
WorkingDirectory=/opt/myapi
ExecStart=/opt/myapi/venv/bin/uvicorn main:app --host 127.0.0.1 --port 8010
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```


| Секция      | Описание                 |
| ----------- | ------------------------ |
| `[Unit]`    | Метаданные и зависимости |
| `[Service]` | Как запускать            |
| `[Install]` | Когда запускать          |

**Nginx выполняет:**

- Раздача статики

# Слайд 17: Конфигурация Nginx

```nginx
# /etc/nginx/sites-available/mysite

server {
    listen 80;                          # Порт
    server_name mysite.ru;              # Домен
    
    location / {                        # Путь
        proxy_pass http://127.0.0.1:8010;  # Куда проксировать
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

**Структура:**

- `server { }` — блок виртуального хоста
- `location / { }` — правила для путей
- `proxy_pass` — адрес бэкенда

# Слайд 18: Nginx — включение сайта

```bash
# Конфиги сайтов
/etc/nginx/sites-available/   # Доступные (неактивные)
/etc/nginx/sites-enabled/     # Включённые (активные)

# Создать символическую ссылку (включить сайт)
ln -sf /etc/nginx/sites-available/mysite /etc/nginx/sites-enabled/

# Проверить синтаксис конфига
nginx -t

# Перезагрузить Nginx
systemctl reload nginx
```

**Важно:** После изменения конфигов всегда проверяйте `nginx -t` перед перезагрузкой!

# Слайд 19: DNS — привязка домена

**DNS (Domain Name System)** — преобразует домен в IP-адрес.

```
mysite.ru  ──▶  DNS-сервер  ──▶  81.90.182.174
```

**A-запись** связывает домен с IP:


| Тип | Имя | Значение      | TTL |
| --- | --- | ------------- | --- |
| A   | @   | 81.90.182.174 | 300 |
| A   | www | 81.90.182.174 | 300 |


- `@` — корневой домен (mysite.ru)
- `www` — поддомен ([www.mysite.ru](http://www.mysite.ru))
- `TTL` — время жизни записи в секундах

# Слайд 20: HTTPS с Certbot

**Certbot** — автоматическое получение SSL-сертификатов от Let's Encrypt.

```bash
# Установка
apt install -y certbot python3-certbot-nginx

# Получение сертификата
certbot --nginx -d mysite.ru -d www.mysite.ru
```

**Что делает Certbot:**

1. Проверяет владение доменом
2. Получает сертификат от Let's Encrypt
3. Настраивает Nginx для HTTPS
4. Добавляет редирект HTTP → HTTPS
5. Настраивает автообновление

# Слайд 25: Модальные редакторы — что это?

**Модальный редактор** — редактор с разными режимами работы.


| Режим       | Назначение          | Клавиша       |
| ----------- | ------------------- | ------------- |
| **Normal**  | Навигация и команды | `Esc`         |
| **Insert**  | Ввод текста         | `i`, `a`, `o` |
| **Visual**  | Выделение текста    | `v`, `V`      |
| **Command** | Команды редактора   | `:`           |


**Популярные модальные редакторы:**

- **Vim** — классика, есть везде
- **Neovim** — современный форк Vim
- **Helix** — новый, Rust-based

**Зачем учить?**

- Vim установлен на любом Linux-сервере
- Быстрое редактирование конфигов
- Не нужна мышь — только клавиатура

**Выживание (минимум для работы):**

| --------------------- | ------------- |

- `:w` — **w**rite (записать)
- `:q` — **q**uit (выйти)
- `i` — **i**nsert (вставка)
