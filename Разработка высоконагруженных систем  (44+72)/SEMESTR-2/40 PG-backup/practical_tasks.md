# Практические задания по логическому бэкапу PostgreSQL

На основе лекции о логическом бэкапе в PostgreSQL, выполните следующие практические задания для базы данных e-shop (файл data.sql). Предполагается, что база данных называется `eshop`, пользователь `postgres`, хост `localhost`.

## Задание 1: Создание полного бэкапа базы данных в формате plain text

**Описание:** Создайте полный логический бэкап всей базы данных e-shop в формате plain text и сохраните его в файл `full_backup.sql`.

**Разбор:** Команда `pg_dump` без дополнительных опций создает бэкап в формате SQL, который можно восстановить с помощью `psql`. Это включает все таблицы, данные, индексы и т.д.

**Решение:**
```bash
pg_dump -U postgres -h localhost -d eshop -f full_backup.sql
```

## Задание 2: Создание бэкапа только схемы базы данных

**Описание:** Создайте бэкап только схемы (структуры таблиц) базы данных e-shop без данных, в файл `schema_only.sql`.

**Разбор:** Опция `--schema-only` исключает данные, оставляя только DDL-команды для создания таблиц, индексов и т.д.

**Решение:**
```bash
pg_dump -U postgres -h localhost -d eshop --schema-only -f schema_only.sql
```

## Задание 3: Создание бэкапа только данных

**Описание:** Создайте бэкап только данных (без схемы) базы данных e-shop в файл `data_only.sql`.

**Разбор:** Опция `--data-only` включает только INSERT-команды для данных, без создания таблиц.

**Решение:**
```bash
pg_dump -U postgres -h localhost -d eshop --data-only -f data_only.sql
```

## Задание 4: Создание бэкапа конкретной таблицы

**Описание:** Создайте бэкап только таблицы `video_cards` из базы данных e-shop в файл `video_cards_backup.sql`.

**Разбор:** Укажите имя таблицы после имени базы данных. Это создаст бэкап только указанной таблицы.

**Решение:**
```bash
pg_dump -U postgres -h localhost -d eshop -t video_cards -f video_cards_backup.sql
```

## Задание 5: Создание бэкапа в сжатом формате custom

**Описание:** Создайте полный бэкап базы данных e-shop в сжатом custom формате PostgreSQL (формат -Fc) в файл `backup.custom`.

**Разбор:** Формат -Fc позволяет сжатие и является более эффективным для больших баз данных. Восстанавливается с помощью `pg_restore`.

**Решение:**
```bash
pg_dump -U postgres -h localhost -d eshop -Fc -f backup.custom
```

## Задание 6: Восстановление бэкапа из custom формата

**Описание:** Восстановите бэкап из файла `backup.custom` в новую базу данных `eshop_restore`.

**Разбор:** Сначала создайте новую базу данных, затем используйте `pg_restore` для восстановления из custom формата.

**Решение:**
```bash
createdb -U postgres eshop_restore
pg_restore -U postgres -h localhost -d eshop_restore backup.custom
```

## Задание 10: Восстановление бэкапа с очисткой базы данных

**Описание:** Восстановите бэкап из `full_backup.sql` в существующую базу данных `eshop`, предварительно очистив её.

**Разбор:** Опция `--clean` в `pg_dump` добавляет DROP-команды, но для восстановления используйте `psql` с опцией `--clean` если нужно.
