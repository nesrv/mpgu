## 1️⃣ Введение: что такое деплой (10 мин)

### Ключевые вопросы

* Чем деплой отличается от `python manage.py runserver`
* Что такое **production**
* Основные компоненты:

  * сервер
  * web-server
  * application server
  * база данных
  * переменные окружения

### Типовая продакшен-схема

```
Browser → Nginx → Gunicorn / Uvicorn → Python app → DB
```

---

## 2️⃣ PaaS — Platform as a Service (15 мин)

### Что это

PaaS — платформа, где **сервер и инфраструктура скрыты** от разработчика.

### Примеры

* Railway
* Render
* Fly.io
* Heroku

### Как выглядит деплой

```
git push → build → run
```

### Плюсы

✔ минимальный порог входа
✔ быстро
✔ CI/CD часто встроен

### Минусы

✖ меньше контроля
✖ ограничения
✖ цена на росте нагрузки

### Когда использовать

* MVP
* обучение
* демо-проекты

---

## 3️⃣ cPanel-хостинг (15 мин)

### Что это

Классический shared-хостинг с графической панелью.

### Особенности

* Ограниченный доступ к системе
* Часто используется **Passenger**
* Python — не основной язык

### Типичный процесс

1. Загрузить файлы
2. Создать virtualenv
3. Настроить WSGI
4. Подключить домен

### Плюсы

✔ дешево
✔ GUI
✔ без DevOps

### Минусы

✖ плохо масштабируется
✖ устаревший подход
✖ неудобно для FastAPI

### Когда использовать

* простые Django-сайты
* legacy-проекты

---

## 4️⃣ VPS (Virtual Private Server) (20 мин)

### Что это

Полный контроль над сервером (Linux).

### Примеры провайдеров

* DigitalOcean
* Hetzner
* Amazon EC2

### Типовая архитектура

```
Nginx
Gunicorn / Uvicorn
PostgreSQL
Redis
systemd
```

### Плюсы

✔ гибкость
✔ дешевле PaaS при росте
✔ продакшен-подход

### Минусы

✖ ручная настройка
✖ ответственность за безопасность

---

## 5️⃣ VPS + Docker (25 мин)

### Зачем Docker

* одинаковое окружение
* быстрый деплой
* изоляция сервисов

### Архитектура

```
Docker
 ├─ web (Django / FastAPI)
 ├─ db (PostgreSQL)
 ├─ redis
 └─ nginx
```

### Что показывать на лекции

* Dockerfile
* docker-compose.yml
* `.env`
* volumes
* ports

### Плюсы

✔ стандарт индустрии
✔ удобно масштабировать
✔ идеально для CI/CD

### Минусы

✖ дополнительный порог входа

### Важно донести

> Docker — **не магия**, а упаковка Linux-процессов

---

## 6️⃣ CI/CD для Python-проектов (20 мин)

### Что такое CI/CD

* **CI** — Continuous Integration
* **CD** — Continuous Deployment

### Типовой pipeline

```
git push
 → tests
 → build
 → deploy
```

### Инструменты

* GitHub Actions
* GitLab CI
* Jenkins

### Пример сценария

1. Push в `main`
2. Запуск тестов
3. Сборка Docker-образа
4. Деплой на VPS

### Что важно подчеркнуть

✔ автодеплой
✔ меньше ошибок
✔ воспроизводимость
