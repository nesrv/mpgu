3. Проектирование API для e-shop

3.1 Модели (shop/models.py)

from django.db import models class VideoCard(models.Model): name = models.CharField(max_length=200) price = models.DecimalField(max_digits=12, decimal_places=2) description = models.TextField(blank=True) created_at = models.DateTimeField(auto_now_add=True) class Meta: db_table = "video_cards" class Cart(models.Model): product = models.ForeignKey(VideoCard, on_delete=models.CASCADE) qty = models.IntegerField(default=1) created_at = models.DateTimeField(auto_now_add=True) class Order(models.Model): total = models.DecimalField(max_digits=12, decimal_places=2) created_at = models.DateTimeField(auto_now_add=True) class OrderItem(models.Model): order = models.ForeignKey(Order, on_delete=models.CASCADE) product_name = models.TextField() qty = models.IntegerField() price = models.DecimalField(max_digits=12, decimal_places=2)

3.2 API Ninja (shop/api.py)

from ninja import NinjaAPI, Schema from .models import VideoCard api = NinjaAPI(urls_namespace="api") class ProductOut(Schema): id: int name: str price: float description: str @api.get("/products", response=list[ProductOut]) def list_products(request): return list(VideoCard.objects.values("id", "name", "price", "description")) @api.get("/products/{product_id}", response=ProductOut) def get_product(request, product_id: int): p = VideoCard.objects.get(id=product_id) return {"id": p.id, "name": p.name, "price": float(p.price), "description": p.description or ""} @api.get("/health") def health(request): return {"status": "ok"}

3.3 Django view для сравнения (shop/views.py)

from django.http import JsonResponse from .models import VideoCard def products_django(request): data = list(VideoCard.objects.values("id", "name", "price", "description")) return JsonResponse({"products": data})

5. Экспериментальная часть: Gunicorn (WSGI и ASGI)

5.1 WSGI: разбор параметров

gunicorn config.wsgi:application \ --bind 127.0.0.1:8000 \ --workers 3 \ --threads ?
ПараметрОписание
--bindАдрес и порт
--workersЧисло процессов. Итого: workers × threads запросов
--threadsПотоков на worker (по умолчанию 1)

Итого при workers=3, threads=1: 3 одновременных запроса.

5.2 ASGI (Gunicorn + UvicornWorker)

gunicorn config.asgi:application \ --bind 127.0.0.1:8000 \ --workers 4 \ -k uvicorn.workers.UvicornWorker
  • config.asgi:application — точка входа ASGI (async)
  • -k uvicorn.workers.UvicornWorker — worker-класс Uvicorn
  • ASGI даёт существенно больший RPS при той же нагрузке

📊 Выводы

  • Django Ninja vs обычный Django view — Ninja даёт меньший overhead и обычно более высокий RPS.
  • Workers и threads — для WSGI: workers = CPU × 2 + 1. Рост threads до 2–4 полезен, дальше — GIL ухудшает latency.
  • Оптимальная конфигурация — для 2 vCPU: 5 workers, 2 threads, max-requests для стабильности.
  • ASGI (Gunicorn + UvicornWorker) — существенно выше RPS, рекомендуется для API на Django Ninja.
